# AI Vision Chat 升级为 Realtime Multimodal Agent 改造计划

## 项目背景

当前项目已经具备：

- Step-1o-Audio 实时语音
- Step-3.7-Flash 图像理解
- WebSocket 双向通信
- 摄像头连续采样
- EventBus
- CostController
- AgentOrchestrator

但当前语音与视觉链路相互独立：

```text
Camera -> Vision -> Browser

Mic -> Realtime -> Browser
```

目标升级为：

```text
Camera -> Vision -> Scene Memory
                                 \
                                  -> Agent Router -> Realtime Session
                                 /
Mic ----------------------------
```

使 AI 能够真正理解：

- 当前画面
- 最近发生的事件
- 用户语音上下文
- 历史视觉记忆

实现类似 Gemini Live / ChatGPT Voice 的实时多模态交互体验。

# Phase 1：Scene Memory（最高优先级）

## 新增文件

```text
scene_memory.js
```

## 功能

维护当前视觉场景状态。

保存：

```js
{
  timestamp,
  location,
  people,
  objects,
  summary,
  recentActions
}
```

示例：

```js
{
  location: "office",
  objects: ["laptop","cup","keyboard"],
  people: 1,
  summary: "用户坐在办公桌前",
  recentActions: [
    "用户拿起杯子"
  ]
}
```

## API

实现：

```js
update(scene)
getContext()
recordAction(action)
hasSignificantChange(scene)
```

# Phase 2：Vision Context Injection

## 修改文件

```text
vision_processor.js
realtime_handler.js
```

## 目标

将视觉分析结果同步到实时语音会话。

当前：

```text
Vision -> Browser
```

改造：

```text
Vision -> SceneMemory
       -> Realtime Session
```

## 新增接口

realtime_handler.js：

```js
updateVisualContext(context)
```

示例：

```js
{
  objects:["cup","laptop"],
  location:"office",
  recentAction:"user picked up cup"
}
```

## 行为

视觉更新后：

自动刷新 Realtime Session 指令。

例如：

```text
当前环境：

位置：办公室

物体：
- laptop
- keyboard
- cup

最近动作：
- 用户拿起杯子
```

这样用户说：

```text
这是什么？
```

时 Realtime 模型能够理解当前视觉上下文。

# Phase 5：Video Understanding

## 修改文件

```text
vision_processor.js
```

## 当前问题

每张图片独立分析：

```text
Frame1
Frame2
Frame3
```

无法理解连续动作。

------

## 新增

维护：

```js
lastScenes[]
```

保存最近：

```text
5~10秒视觉摘要
```

------

分析时提交：

```text
历史视觉摘要

+
当前帧
```

给 Step-3.7 Flash。

------

目标

让模型理解：

```text
用户刚刚坐下
用户刚刚拿起杯子
用户正在喝水
```

而不是仅理解当前图片。

# Phase 6：Adaptive FPS

## 修改文件

```text
cost_controller.js
```

## 新增

帧差检测：

```js
calculateFrameDifference()
```

## 策略

画面变化小：

```text
0.2 FPS
```

即：

```text
5秒分析一次
```

------

画面变化大：

```text
1 FPS
```

即：

```text
1秒分析一次
```

------

目标

减少：

```text
50%~90%
```

视觉调用成本。
