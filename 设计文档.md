## 2. 成本控制策略

### 想到的策略 vs 实际采用的策略

| 策略 | 描述 | 想到 | 采用 | 说明 |
|------|------|------|------|------|
| **智能帧采样** | 画面无变化时自动降低帧率（1 fps → 0.2 fps） | ✅ | ✅ | `cost_controller.FrameRateController` — 检测连续相同帧，3 次无变化后降频 |
| **画面分辨率压缩** | 将 1080p 帧压缩到 640px/512px 后再送 Vision API | ✅ | ✅ | `vision_processor._resize_image` — 缩小后 JPEG quality 70%，token 减少约 85% |
| **Low Detail 模式** | GPT-4 Vision 使用 `detail: "low"` 而非默认 high | ✅ | ✅ | 每帧仅 85 tokens（vs high 的 170+ tokens） |
| **静音检测** | 音频能量低于阈值时跳过 Whisper STT 调用 | ✅ | ✅ | `SilenceDetector` — 计算 PCM RMS，低于 threshold=500 跳过 |
| **对话 Token 预算** | 跟踪累计 Token，超限时自动摘要旧消息 | ✅ | ✅ | `ConversationBudget` — max_tokens=4096，超限触发 deepagents summarization |
| **TTS 模型选择** | 使用 `tts-1` 而非 `tts-1-hd`（更快更便宜） | ✅ | ✅ | `tts-1` 成本 $0.015/1K chars，速度更快 |
| **Whisper 语言限制** | 指定 `language="zh"` 减少识别歧义 | ✅ | ✅ | 减少重试和错误识别导致的浪费 |
| **WebSocket 复用** | 单一长连接避免重复建连开销 | ✅ | ✅ | 使用 WebSocket 持续连接 + ping/pong 保活 |
| **音频编码优化** | 使用 16kHz 单声道 PCM 而非更高采样率 | ✅ | ✅ | `record` 配置 sampleRate=16000, numChannels=1 |
| **本地预过滤** | 在 Flutter 端做基础 VAD 后再发送 | ✅ | ❌ | Flutter 端缺少高效的 VAD 库，延迟 MVP |
| **TTS 缓存** | 缓存常见短语的 TTS 结果 | ✅ | ❌ | 对话场景重复率低，收益有限 |
| **多模型分层** | 简单问题用快/便宜模型，复杂问题用强模型 | ✅ | ❌ | 增加架构复杂度，延迟 MVP |
| **帧差异哈希** | 用感知哈希检测画面变化，而非直接比较 Base64 | ✅ | ❌ | Base64 比较已足够，感知哈希增加计算开销 |
| **请求合并** | 多帧合并为单次 Vision 请求 | ✅ | ❌ | 实时场景需要低延迟，不适合批处理 |

### 成本估算（理论值）

假设用户每天使用 30 分钟：

| 项目 | 单价 | 单次用量 | 单次费用 | 日费用 | 月费用 |
|------|------|---------|---------|-------|-------|
| GPT-4 Vision (low detail) | $2.50/1M tokens | ~170 tokens/帧 | $0.000425 | (0.5帧/秒×1800秒)×$0.000425 = $0.38 | ~$11.5 |
| Whisper STT | $0.006/minute | 3秒/次 | $0.0003 | ~$0.18 | ~$5.4 |
| TTS (`tts-1`) | $0.015/1K chars | ~50 chars/次 | $0.00075 | ~$0.45 | ~$13.5 |
| **总计** | | | | **~$1.01** | **~$30.4** |

> 实际使用中，智能帧采样和静音检测可再降低 40-60% 费用。

### 数据流（一次典型对话回合）

```
1. [Flutter] Camera → Canvas 截帧 → JPEG Base64
2. [Flutter] MediaRecorder → 3s PCM 16kHz 音频
3. [Flutter] WebSocket → {"type":"frame", "data":"<base64>"}
4. [Flutter] WebSocket → {"type":"audio", "data":"<base64>"}
5. [Backend] 音频 → SilenceDetector (跳过静音)
6. [Backend] 音频 → Whisper API → 文本
7. [Backend] 帧 → FrameRateController (1 fps, 静态降频)
8. [Backend] 帧 → Vision API → 画面描述文本
9. [Backend] Agent: 融合(画面描述 + 语音文本) → deepagents 推理 → 回复
10. [Backend] TTS API → MP3 音频
11. [Backend] WebSocket → {"type":"assistant_message", "text":"..."}
12. [Backend] WebSocket → 二进制音频 MP3
13. [Flutter] 显示文字气泡 + 播放语音
```

## 4. 模块接口说明

### 后端模块

| 模块 | 文件 | 主要接口 | 说明 |
|------|------|---------|------|
| WebSocket 网关 | `websocket_handler.py` | `ConnectionManager` | 会话管理、消息路由 |
| 视觉处理 | `vision_processor.py` | `process_frame()`, `analyze_frame()` | 帧采样 → 压缩 → Vision API |
| 音频处理 | `audio_processor.py` | `process_audio_chunk()`, `transcribe_audio()`, `synthesize_speech()` | 音频 → Whisper → TTS |
| Agent 编排 | `agent_orchestrator.py` | `process_user_input()`, `get_agent()` | deepagents 对话推理 |
| 成本控制 | `cost_controller.py` | `FrameRateController`, `SilenceDetector`, `ConversationBudget` | 资源管理 |

### Flutter 前端模块

| 模块 | 文件 | 主要类 | 说明 |
|------|------|--------|------|
| 主入口 | `main.dart` | `VisionChatApp` | MaterialApp 配置 |
| 对话界面 | `screens/chat_screen.dart` | `ChatScreen` | 主界面，集成所有服务 |
| 相机服务 | `services/camera_service.dart` | `CameraService` | 摄像头初始化、帧采集 |
| 音频服务 | `services/audio_service.dart` | `AudioService` | 麦克风录音、音频流 |
| WebSocket 服务 | `services/websocket_service.dart` | `WebSocketService` | 连接管理、消息收发 |
| 语音播放 | `services/audio_player_service.dart` | `AudioPlayerService` | TTS 音频播放 |
| 权限服务 | `services/permission_service.dart` | `PermissionService` | 摄像头/麦克风权限 |
| 消息模型 | `models/message.dart` | `ChatMessage` | 对话消息数据模型 |
| 消息气泡 | `widgets/message_bubble.dart` | `MessageBubble` | 聊天气泡组件 |
| 控制栏 | `widgets/control_bar.dart` | `ControlBar` | 底部控制按钮 |

## 5. 配置项参考

| 配置 | 默认值 | 说明 |
|------|--------|------|
| `MAX_FPS` | 1.0 | 视频帧最大采样率（帧/秒） |
| `MIN_FPS` | 0.2 | 静态场景最小采样率（帧/秒） |
| `SILENCE_THRESHOLD` | 500 | 静音检测 RMS 阈值（0-32768） |
| `MAX_TOKENS` | 4096 | 对话 Token 预算 |
| `FRAME_RESIZE` | 512 | 画面压缩边长（像素） |
| `JPEG_QUALITY` | 70 | JPEG 压缩质量（0-100） |
| `AUDIO_CHUNK_MS` | 3000 | 音频分段时长（毫秒） |
| `AUDIO_SAMPLE_RATE` | 16000 | 音频采样率（Hz） |
| `WS_RECONNECT_DELAY` | 1000 | WebSocket 重连初始延迟（毫秒） |
| `WS_MAX_RECONNECT_DELAY` | 30000 | 最大重连延迟（毫秒） |
