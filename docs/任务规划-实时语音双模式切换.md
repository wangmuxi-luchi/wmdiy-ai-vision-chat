# 实时语音双模式切换 — 任务规划

## 目标

在前端支持两种语音交互模式切换：

| 模式 | 方案 | 流程 | 特点 |
|------|------|------|------|
| 管道式语音聊天 | ASR → LLM → TTS | 语音转文字 → 文本大模型 → 浏览器 TTS 朗读 | 经济实惠，当前主方案 |
| 实时端到端语音聊天 | `step-1o-audio` | 音频二进制 → 端到端模型 → AI 语音流 → 播放 | 快，一步到位，成本较高 |

---

## 技术可行性

Flutter 的 `web_socket_channel` 原生支持文本 + 二进制消息，对接完全没问题：

| 点 | Flutter 支持情况 |
|----|----------------|
| 发送二进制 PCM/WAV | ✅ `Uint8List` → `_channel.sink.add(Uint8List)` |
| 接收二进制音频流 | ✅ `stream.listen((data) { if (data is Uint8List) { ... } })` |
| 浏览器音频播放 | ✅ JS interop 调用 `AudioContext.decodeAudioData()` |
