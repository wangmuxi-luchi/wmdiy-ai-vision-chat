# AI 视觉对话助手 — PPT 文字稿

---

## 第 1 页 · 封面

**标题：AI 视觉对话助手**

**副标题：** 让 AI 看见你、听见你，实时语音对话

**项目定位：** 基于阶跃星辰大模型的多模态实时交互系统，支持摄像头视觉理解 + 语音输入输出，跨平台（Web / Android / iOS）部署

---

## 第 2 页 · 项目概述

**核心能力：**

- 📷 **视觉感知** — 通过摄像头实时捕获画面，AI 分析画面内容融入对话上下文
- 🎤 **语音输入** — 麦克风实时采集语音，支持 ASR 语音转文字
- 🔊 **语音输出** — AI 回复文本自动朗读（TTS），语速可调
- 💬 **多模态融合** — 画面描述 + 语音文本融合后送入大模型，生成自然口语化回复
- ⚡ **实时交互** — WebSocket 双向通信，低延迟对话体验

**一句话总结：** AI 像一个有眼睛有耳朵的朋友，能看能听能说，和你自然聊天。

## 第 3 页 · 技术架构

**技术栈：**

| 层级 | 技术 | 说明 |
|------|------|------|
| 大模型 | 阶跃星辰 `step-3.7-flash` | 视觉理解 + 对话推理 |
| 语音识别 | 腾讯云 ASR | 前端语音转文字 |
| 语音合成 | 浏览器 Web Speech API | 前端本地 TTS，零成本 |
| 后端 | Node.js + Express + WebSocket | 消息路由、会话管理 |
| 前端 | Flutter 3.x | 跨平台（Web/Android/iOS） |
| 图片处理 | sharp | 摄像头帧压缩 |

**架构图：**

```
Flutter 前端 ──WebSocket──▶ Node.js 后端 ──HTTPS──▶ 阶跃星辰 API
    │                                                      │
    ├── 摄像头截帧 (JPEG)                                  ├── step-3.7-flash 视觉分析
    ├── 麦克风录音 (语音)                                  ├── step-3.7-flash 对话推理
    └── 文字输入                                           └── 返回 AI 回复文本
```

## 第 4 页 · 后端架构

**模块划分：**

| 模块 | 职责 |
|------|------|
| `server.js` | 服务入口，Express + WebSocket 启动 |
| `websocket_handler.js` | 会话管理 + 消息路由（frame / audio / text） |
| `vision_processor.js` | 图像压缩 → 调用 Vision API 分析画面 |
| `agent_orchestrator.js` | 对话编排，融合画面描述 + 语音文本 |
| `cost_controller.js` | Token 预算管理 + 静音检测 |

**数据流：**

```
摄像头截帧 → sharp 压缩至 512px → 阶跃星辰 Vision API → 画面描述
    ↓
麦克风录音 → 腾讯云 ASR → 用户语音文本
    ↓
画面描述 + 语音文本 → 融合 → 阶跃星辰 LLM → AI 回复
    ↓
AI 回复文本 → 前端聊天展示 + 浏览器 TTS 朗读
```

**关键设计决策：** 后端 TTS 已禁用，由前端浏览器本地 TTS 接管，节省 Token 成本且响应更快。

## 第 5 页 · 语音交互方案对比

**两套方案：**

| | 方案 A：实时语音聊天 | 方案 B：管道式语音聊天 |
|------|------|------|
| **模型** | `step-1o-audio` | `step-3.7-flash` |
| **流程** | 端到端，音频输入 → 音频输出 | 分步：ASR → LLM → TTS |
| **中间环节** | 模型内部一体化处理 | 语音转文字 → 文本大模型 → 文字转语音 |
| **速度** | ⚡ 快（一步到位） | 🐢 较慢（多步串联） |
| **成本** | 💰 较高 | 💸 经济实惠 |
| **适用场景** | 实时语音流，需要快速交互 | 文字对话，需要控制成本 |

**方案 A：实时端到端（`step-1o-audio`）**

```
麦克风 PCM 音频 ──WebSocket──→ 后端 ──→ 阶跃星辰 Realtime API
                                                  │
                                    内置 VAD（语音端点检测）
                                    内置 ASR（语音转文字）
                                    内置 LLM（对话生成）
                                    内置 TTS（文字转语音）
                                                  │
                    AI 语音流 ←──WebSocket── 后端 ←┘
```

**方案 B：管道式 ASR + LLM + TTS**

```
麦克风录音 ──→ 腾讯云 ASR ──→ 用户语音文本
                                  ↓
                           阶跃星辰 LLM ──→ AI 回复文本
                                  ↓
                          浏览器 Web Speech API ──→ AI 语音朗读
```

**当前策略：** 方案 B 为主（经济实惠），方案 A 代码已就绪可按需启用。

## 第 6 页 · 四大核心功能

**1. 语音转文字（ASR）**
- 腾讯云实时语音识别，支持流式识别
- 说完一句话自动结束，结果自动发送

**2. 图像传输（多选模式）**
- **语音触发发送** — 用户说话时自动捕获一帧画面
- **定时发送** — 按配置间隔（秒）自动发送画面
- 两者可同时启用，语音触发后自动重置定时器

**3. 朗读（TTS）**
- 浏览器本地 Web Speech API 朗读 AI 回复
- 语速可配置（0.1 ~ 9.9x），手动输入

**4. 实时对话**
- 画面 + 语音融合后送入大模型
- 口语化中文回复，2-4 句话，自然流畅

## 第 7 页 · 关键设计亮点

**1. 多选图像传输配置**
- 从单选改为多选：语音触发 + 定时发送可同时启用
- 定时间隔手动输入，最小 1 秒
- 语音触发后自动重置定时器，避免冗余发送

**2. 后端 TTS 移除**
- 原方案：后端调用 `step-tts-mini` 合成 MP3 → 前端播放
- 现方案：前端浏览器 Web Speech API 直接朗读
- 效果：节省 Token 消耗，响应更快（无需等待 TTS 合成）

**3. 零空闲消耗**
- 后端完全事件驱动，无定时任务、无轮询
- 前端不发消息，后端零 Token 消耗

**4. 配置持久化**
- Chrome 固定端口 `--web-port 8080`，避免 localStorage 因端口变化丢失

## 第 9 页 · 演示效果

**使用场景：**

1. 打开应用 → 浏览器请求摄像头和麦克风权限
2. AI 看到画面 → 自动分析并描述
3. 用户说话 → 语音转文字 → 融合画面上下文 → AI 回复

**运行方式：**

```bash

npm start
# 终端 2：前端（Chrome）
cd frontend_kotlin && flutter run -d chrome --web-port 8080
```
